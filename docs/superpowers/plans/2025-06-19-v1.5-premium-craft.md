# MoodLab V1.5 Premium Craft Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship V1.5 premium craft — onboarding to first export, face-aware skin regions, full-res Skia export, photo LUT thumbnails, and richer LUT grades.

**Architecture:** Keep JS/Skia pipeline (no native RenderCore yet). Extract shared shader uniform building for preview + export parity. Face region uses orientation heuristics now, ML Kit hook later. Onboarding persists via AsyncStorage and routes before tabs.

**Tech Stack:** Expo SDK 56, react-native-skia 2.6, @moodlab/lut-engine, @moodlab/shared, AsyncStorage, expo-sharing, expo-file-system/legacy

## Global Constraints

- MoodLab is a **creator LUT photo editor** — not mood tracking
- Every slider/control must affect preview (honest rendering rule)
- North star: import → mood → export in **≤60 seconds** for new users
- Pro gating stays on premium LUTs; free tier must produce a good first export
- Native RenderCore (CIColorCube / AGSL) is V2 — V1.5 uses Skia offscreen at export preset dimensions
